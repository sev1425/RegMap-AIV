# Contributing to RegMap AI

First off, thank you for considering contributing to RegMap AI! It's people like you that make RegMap AI such a great tool for the compliance community.

## 1. Where do I go from here?

If you've noticed a bug or have a feature request, make one! It's generally best if you get confirmation of your bug or approval for your feature request this way before starting to code.

## 2. Fork & create a branch

If this is something you think you can fix, then fork RegMap AI and create a branch with a descriptive name.

## 3. Get the test suite running

Make sure you have the environment set up and the tests are passing.
```bash
cd backend
pytest
```

## 4. Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help; everyone is a beginner at first.

## 5. Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with RegMap AI's master branch.
Then create a pull request with a clear list of what you've done. When you submit a PR, the CI suite will automatically run.
